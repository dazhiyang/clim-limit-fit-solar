# Data Recreation Instructions

The scripts in this repository (located in the `Code/` folder) expect a data directory named `Data/` in the project root. This document explains how to recreate that directory using the contents of `SampleData/` or by downloading additional datasets.

## 1. Quick Start: Use Sample Data
To run the representative cases (e.g., `4.8.FalsePositive.R` or `4.9.FalseNegative.R`) using the provided sample files, you can simply rename or link the `SampleData/` folder to `Data/`:

```bash
# From the project root
mv SampleData Data
```

Note: Some scripts may look for specific station files not included in this sample set.

## 2. Full Data Directory Structure
To recreate the full environment, your `Data/` folder should look like this:

```text
Data/
├── BSRN/                   # Full BSRN station archive (NetCDF or processed)
├── BSRNtxt/                # Tab-separated station files (e.g., gur.txt, gan.txt)
├── McClear/                # Clear-sky irradiance data
├── Regime2_ERL_only.txt    # Discrepancy cases for Regime 2
├── Regime6_New_only.txt    # Discrepancy cases for Regime 6
├── cluster.csv             # Station-to-regime mapping
├── cluster_parameter.csv   # Model parameters for each regime
├── extraction_features.csv # Statistical features for each station
└── rejection_summary.csv   # Global QC rejection statistics
```

## 3. How to Obtain the Full Dataset
Due to the large file sizes (total > 1GB), only a subset of the data is provided in this repository.

### BSRN Data
The primary radiation data originates from the **Baseline Surface Radiation Network (BSRN)**.
- **Source**: [BSRN Data Archive](https://bsrn.awi.de/)
- **License**: Refer to the BSRN data policy.

### Processed Files
The files `Regime2_ERL_only.txt` and `Regime6_New_only.txt` are intermediate results generated by our analysis pipeline. If you need the full version of these files for large-scale replication, please contact:

## 4. Troubleshooting
If a script fails with a "File not found" error, ensure you have:
1. Created the `Data/` folder.
2. Placed the required `.csv` and `.txt` files directly inside `Data/`.
3. Populated the `BSRNtxt/` subfolder with the necessary station files (e.g., `gur.txt`, `gan.txt`).
